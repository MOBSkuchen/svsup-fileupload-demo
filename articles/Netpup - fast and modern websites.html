Netpup - fast and modern websites

Creating a basic website is easy. Or at least it should be. But nothing works without the HTTP-Server.

Netpup offers a good middle ground between high performance, stability and functionality. It is written in rust and uses minimalism as a feature.

Where other frameworks require the user to either write the code themselves, using the server as a library, netpup just uses a simple config to set routes and such.

By using minimalism, netpup may miss a large group of users who require large apps with complex features and APIs, but netpup is not for those who seek an industrial solution. It is for those who overengineered in the past and can appreciate simplicity.

This is an example of a netpup config (toml):

ip = "95.216.143.121"
port = 5000
cwd = "./data"

# Make sure that routes with '*' come last
[routes.resources]
methods = ["GET"]
url = "/r/*"
path = "resources/*"

[routes.main]
methods = ["GET"]
url = "/*"
path = "index.html"
content_type = "text/html"

As you can see, it isn't very much, but its fine to power something like a mini-tool, personal page, article, blog, whatever.

And the best part is that its very easy to set up. Just run

cargo install netpup

Create a config like shown before, and run it using

netpup --start
